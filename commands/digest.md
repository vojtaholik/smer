# Daily smer digest

Start with `smer brief --since 1d --json`, then read today's chronology with `smer timeline --day $(date +%F) --json`. Inspect candidate event ids with `smer show` and use targeted searches only for missing context. Return chronological project blocks, wins and fixes with event-id citations, approximate time by project, open loops, and one optional build-in-public draft based only on cited events. State that time is inferred from event density. Re-check output for secrets, private URLs, email addresses, customer data, and identifying details. Never invent outcomes or publish automatically.
