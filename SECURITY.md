# Security Policy

## Supported versions

smer is early-stage software. Security fixes are applied to the latest release
and the `main` branch.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability or include sensitive
captured data in any report. Use GitHub's private vulnerability reporting flow
from the repository's **Security** tab.

Include the affected version, impact, reproduction steps, and a minimal
synthetic example. You should receive an acknowledgment within seven days.

Reports involving exposed credentials should also follow the provider's token
revocation guidance immediately. smer's redaction is defense in depth, not a
substitute for revoking a leaked credential.
